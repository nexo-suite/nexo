# Internationalization (i18n)

## Overview

Nexo uses [Paraglide.js](https://inlang.com/m/gerre34r/library-inlang-paraglideJs) for compile-time internationalization. Each app has its own set of messages compiled to tree-shakeable functions — no runtime loading, no bundle overhead.

Supported languages: **English** (source), **Deutsch**, **Türkçe**.

---

## Architecture

```
apps/<app>/
├── project.inlang/settings.json   ← Paraglide project config
├── messages/
│   ├── en.json                    ← source language (English)
│   ├── de.json                    ← German translations
│   └── tr.json                    ← Turkish translations
└── src/lib/
    ├── paraglide/                 ← generated (don't edit manually)
    │   ├── messages.js            ← compiled message functions
    │   ├── messages/              ← per-language message modules
    │   └── runtime.js             ← language tag resolution
    └── i18n.ts                    ← Paraglide i18n config (createI18n)
```

The shared `packages/i18n` package provides:

- `Language` type (`'en' | 'de' | 'tr'`)
- `detectLanguage(acceptLanguageHeader)` — parses Accept-Language to best match

## How it works

### Vite plugin

Each app's `vite.config.ts` includes the Paraglide plugin:

```typescript
import { paraglide } from '@inlang/paraglide-sveltekit/vite';

plugins: [
	paraglide({ project: './project.inlang', outdir: './src/lib/paraglide' })
	// ...
];
```

The plugin compiles `messages/*.json` → `src/lib/paraglide/messages.js` on every save during dev.

### Using messages in components

```svelte
<script lang="ts">
	import * as m from '$lib/paraglide/messages';
</script>

<h1>{m.page_title()}</h1><p>{m.greeting({ name: user.name })}</p>
```

Messages are functions that return strings. Parameters use `{placeholder}` syntax in the JSON.

### Language resolution

In each app's `hooks.server.ts`, language is resolved in this order:

1. **User preference** — `auth.user_preferences.language` (if not `'auto'`)
2. **Accept-Language header** — parsed by `@nexo/i18n` detectLanguage()
3. **Fallback** — `'en'`

The resolved language is set via Paraglide's `i18n.handle()` middleware.

## Adding a new message

1. Add the key to `messages/en.json`:

   ```json
   { "new_key": "Hello {name}, welcome back!" }
   ```

2. Add translations to `de.json` and `tr.json`:

   ```json
   { "new_key": "Hallo {name}, willkommen zurück!" }
   ```

   ```json
   { "new_key": "Merhaba {name}, tekrar hoş geldin!" }
   ```

3. Use it in a component:
   ```svelte
   {m.new_key({ name: user.name })}
   ```

The Vite plugin recompiles immediately during dev. Type checking via JSDoc annotations in the generated `.js` (requires `allowJs: true` in tsconfig).

---

## Message conventions

- Keys use `snake_case`: `nav_home`, `expense_form_title`, `error_not_found`
- Group by feature/page: `login_heading`, `login_sub`, `login_error_server`
- Keep messages short and casual — this is a personal app, not enterprise software
- Include personality in empty states and error messages
- Parameters use `{curlyBraces}`

### Tone per language

| Language | Tone                                          |
| -------- | --------------------------------------------- |
| English  | Casual, slightly witty                        |
| German   | Like texting a friend, not filling a Formular |
| Turkish  | Equally informal, natural conversational      |

## Shared error messages (@nexo/errors)

The `@nexo/errors` package provides user-facing error messages that are also i18n-aware:

```typescript
import { userMessage } from '@nexo/errors';

const msg = userMessage('SESSION_EXPIRED', 'de');
// → "Deine Sitzung ist abgelaufen. Meld dich nochmal an."
```

Error codes are defined once; translations for all languages live in the package.

## Adding a new language

1. Add the language tag to each app's `project.inlang/settings.json`:

   ```json
   { "languageTags": ["en", "de", "tr", "fr"] }
   ```

2. Create `messages/fr.json` in each app with all keys translated.

3. Update `packages/i18n/src/types.ts`:

   ```typescript
   export type Language = 'en' | 'de' | 'tr' | 'fr';
   ```

4. Add translations to `packages/errors/src/index.ts`.

5. Update the settings UI language picker in each app.

## TypeScript considerations

Paraglide generates `.js` files with JSDoc annotations (not `.d.ts` files). For `svelte-check-native` and IDE support to work:

- Each app's `tsconfig.json` must have `"allowJs": true`
- The SvelteKit-generated tsconfig already includes `src/**/*.js` in its `include` array
- Type inference comes from JSDoc `@param` and `@returns` annotations in the generated code

## Troubleshooting

**Messages not updating during dev**
The Paraglide Vite plugin watches `messages/*.json`. If changes don't reflect, restart the dev server.

**Type errors on `$lib/paraglide/messages`**
Ensure `allowJs: true` is in the app's `tsconfig.json`.

**Generated output is stale after git pull**
Run `pnpm dev` (or `svelte-kit sync`) to regenerate the Paraglide output.
